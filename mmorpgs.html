Entrando en MMORPGs

Desde que era chico, siempre me gustaron los juegos MMORPGs. Siempre me pregunté cómo funcionaban, así que hice uno. Bueno... un prototipo funcional para ser exacto. Hecho con Java SE, utilizando una arquitectura "Entity Component System".


Existen miles de motores de desarrollo de videojuegos, pero nada mejor que hacerlo de forma nativa en JAVA, de esta forma se pueden presentar mejores retos y desafíos de los que se puede aprender mucho.

Para este juego, utilicé la librería gráfica LibGdx, y una arquitectura de desarrollo de videojuegos llamada “Entity Component System”.

Al ser un juego multijugador, requiere un servidor centralizado. Por lo que utilicé mi Raspberry PI 3 con Ubuntu Server para servirlo. Dicho servidor lo realicé, así como el mismo juego, en Java.

La comunicación entre dichos componentes, fué mediante Sockets nativos de java.


Sistema de scripting

Desarrollé un sistema de scripting para poder darle inteligencia a distintas entidades. En este caso, un NPC “Leñador” el cual solo se encarga de cortar árboles. Utilizando el mecanismo multihilo de pathfinding que implementé, buscan su próximo destino (árbol) y comienzan a talarlo.



Test de stress

Una de las tareas era optimizar el uso del CPU y el flujo de datos entre distintos clientes, ya que el servidor… Era un raspberry. Para ello agregue varios de cientos de “caminantes”, y analicé los recursos utilizados por el servidor.
